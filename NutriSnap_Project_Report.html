Abstract

In today's health-conscious society, tracking dietary intake is essential for maintaining a balanced lifestyle. However, manual food logging is tedious and often inaccurate. This project presents NutriSnap, an AI-powered web application that automates food recognition and nutrition tracking using deep learning and computer vision techniques.

NutriSnap allows users to simply take a photograph of their food, after which the system automatically identifies the food item using a Convolutional Neural Network (CNN) trained on the Food-101 dataset comprising 101 food categories. The system employs a dual-recognition strategy — utilizing the Hugging Face Vision Transformer (ViT) API as the primary recognizer with the local CNN model as a fallback, ensuring reliability even without internet connectivity.

Upon successful food identification, the system retrieves detailed nutritional information (calories, protein, carbohydrates, and fat) from the USDA FoodData Central API, with a local CSV-based database as a secondary source. Users can log their meals, view daily/weekly/monthly nutrition summaries on a personalized dashboard, compete with friends on a health-score leaderboard, and receive AI-driven meal recommendations based on their dietary patterns.

The application is built using the Flask web framework following the MVC (Model-View-Controller) architecture with Service and Repository layers, ensuring clean separation of concerns and maintainability. Security is implemented through JWT (JSON Web Token) based authentication with password hashing.

Keywords: Food Recognition, Deep Learning, Convolutional Neural Network, Nutrition Tracking, Computer Vision, Flask, Food-101, Vision Transformer, REST API

Chapter 1: Introduction

1.1 Project Overview

NutriSnap is an AI-powered web application designed to simplify nutrition tracking through automated food recognition. Instead of manually searching for food items and entering nutritional data, users can simply photograph their meal, and the system automatically identifies the food and retrieves its complete nutritional profile including calories, protein, carbohydrates, and fat content.

The application integrates deep learning models for image classification, external APIs for accurate nutrition data, and a comprehensive dashboard for tracking dietary patterns over time. It also features social elements such as a friends system, health-score leaderboard, and personalized meal recommendations.

1.2 Problem Statement

In the modern era, lifestyle diseases such as obesity, diabetes, and cardiovascular disorders are on the rise, largely due to poor dietary habits. While nutrition tracking applications exist (such as MyFitnessPal), they require users to manually search and log every food item — a process that is:

  • Time-consuming: Users must type food names, select portion sizes, and verify nutritional data for each meal.
  • Error-prone: Manual entry leads to inaccurate logging due to estimation errors and incomplete data entry.
  • Inconvenient: The friction of manual logging causes most users to abandon food tracking within weeks.

There is a need for an automated system that can recognize food from images and instantly provide accurate nutritional information, thereby reducing the effort required for dietary tracking.

1.3 Objectives

  1. To develop a food recognition system using Convolutional Neural Networks (CNN) capable of identifying 101 food categories from images.
  2. To integrate external APIs (Hugging Face and USDA) for enhanced accuracy in food recognition and nutrition data retrieval.
  3. To build a web-based dashboard for visualizing daily, weekly, and monthly nutrition summaries.
  4. To implement a leaderboard and recommendation system to encourage healthy eating habits.
  5. To ensure secure user authentication using JWT-based token authentication.
  6. To follow clean software architecture principles (MVC + Service-Repository pattern).

1.4 Scope of the Project

The system can recognize 101 food categories from the Food-101 dataset. It provides nutritional tracking for individual users with features including meal logging, dashboard analytics, social leaderboards, friend analytics, and meal recommendations. The application is designed as a responsive web application accessible via both desktop and mobile browsers.

Chapter 2: Literature Survey

2.1 Food Recognition Using Deep Learning

Bossard et al. (2014) introduced the Food-101 dataset, comprising 101,000 images across 101 food categories. This dataset has become the standard benchmark for food recognition research, achieving baseline accuracy of 50.76% using Random Forests. Subsequent deep learning approaches have significantly improved upon this, with modern CNN architectures achieving over 90% accuracy.

2.2 Convolutional Neural Networks (CNNs)

CNNs are a class of deep learning models specifically designed for processing visual data. They use convolutional layers to automatically learn hierarchical features from images — from simple edges in early layers to complex food textures and shapes in deeper layers. Key architectures include VGGNet, ResNet, InceptionNet, and EfficientNet, all of which have been successfully applied to food recognition tasks.

2.3 Vision Transformers (ViT)

Dosovitskiy et al. (2020) introduced the Vision Transformer, which applies the Transformer architecture (originally designed for NLP) to image classification. ViT divides an image into fixed-size patches, linearly embeds each patch, and processes them using standard Transformer encoder blocks. The nateraw/food model used in NutriSnap is a ViT fine-tuned on the Food-101 dataset, offering superior accuracy compared to traditional CNNs.

2.4 Nutrition APIs and Databases

The USDA FoodData Central database is the most comprehensive publicly available food composition database, maintained by the United States Department of Agriculture. It provides detailed nutritional information for thousands of food items, including macronutrients, vitamins, and minerals. The database is accessible through a free REST API.

Chapter 4: System Design

4.1 System Architecture

NutriSnap follows the MVC (Model-View-Controller) architecture enhanced with Service and Repository layers for clean separation of concerns:

  • Model Layer: Defines database schema using SQLAlchemy ORM (User, FoodLog, NutritionInfo, Friendship).
  • View Layer: HTML templates rendered by Flask's Jinja2 template engine with CSS/JavaScript frontend.
  • Controller Layer: Flask Blueprints that handle HTTP requests and route them to appropriate services.
  • Service Layer: Contains business logic — ML inference, nutrition lookup, leaderboard scoring, recommendations.
  • Repository Layer: Encapsulates all database queries using SQLAlchemy, providing clean data access methods.

4.4 Food Recognition Pipeline

The food recognition system uses a dual-model strategy with fallback mechanism:

  1. Step 1 — Image Upload: User captures or uploads a food image via the web interface.
  2. Step 2 — Primary Recognition (Hugging Face API): Image bytes are sent to the Hugging Face nateraw/food Vision Transformer model via HTTP POST. If confidence ≥ 60%, the result is accepted.
  3. Step 3 — Fallback Recognition (Local CNN): If the API fails or returns low confidence, the local CNN model processes the image: resize to 200×200, normalize pixels to [0,1], predict across 101 classes.
  4. Step 4 — Nutrition Lookup: The identified food name is queried against the USDA API. If unavailable, local CSV data is used.
  5. Step 5 — Result Display: Food name, confidence score, and full nutrition breakdown are returned to the user.

5.3 ML Service — Food Recognition (Core Feature)

The MLService class is the heart of the application. It implements a dual-model recognition strategy:

5.3.1 Primary Recognition — Hugging Face API

The system first sends the food image to the Hugging Face Inference API, which hosts the nateraw/food Vision Transformer model. This model was fine-tuned on the Food-101 dataset.

response = requests.post( 'https://api-inference.huggingface.co/models/nateraw/food', headers=headers, data=image_bytes, timeout=15 )

5.3.2 Fallback — Local CNN Model

If the API fails or returns low confidence (<60%), the local model is used:

# Load and preprocess image img = image.load_img(image_path, target_size=(200, 200)) img_array = image.img_to_array(img) img_array = np.expand_dims(img_array, axis=0) img_array = img_array / 255.0 # Normalize to [0, 1] # Predict pred = cls.model.predict(img_array) top_indices = np.argsort(pred[0])[-3:][::-1] # Top 3 food_name = cls.label[top_indices[0]] confidence = float(pred[0][top_indices[0]])

5.4 Authentication System

User authentication is implemented using JWT (JSON Web Tokens). The security module provides three key functions:

  • generate_token(user_id): Creates a JWT containing the user ID, issue time, and expiration (24 hours).
  • token_required decorator: Validates the JWT from the Authorization header before allowing access to protected endpoints.
  • role_required(roles) decorator: Provides role-based access control (USER, TRAINER, DOCTOR).

Passwords are hashed using Werkzeug's generate_password_hash() function (bcrypt-based) before storage. Plain-text passwords are never stored in the database.

5.7 Recommendation Engine

The recommendation service analyzes the user's 7-day dietary averages and identifies nutritional gaps. Based on identified gaps, it suggests meals from a curated database of 18 healthy meals tagged with categories (high_protein, low_calorie, low_carb, healthy_fat).

Chapter 7: Conclusion & Future Scope

7.1 Conclusion

NutriSnap successfully demonstrates the practical application of deep learning and computer vision in the domain of health and nutrition. The system achieves its primary objective of automating food recognition and nutrition tracking, significantly reducing the manual effort traditionally required for dietary monitoring.

Key achievements of this project include:

  • Successful implementation of a dual-model food recognition system capable of identifying 101 food categories with a confidence threshold mechanism.
  • Integration of the Hugging Face Vision Transformer API as the primary recognizer with a local CNN model as a reliable fallback.
  • Development of a comprehensive nutrition tracking dashboard with daily, weekly, and monthly analytics.
  • Implementation of a gamified leaderboard system to encourage healthy eating among friends.
  • Creation of an AI-driven recommendation engine that analyzes dietary patterns and suggests nutritionally appropriate meals.
  • Secure authentication system using JWT tokens and password hashing.
  • Clean, maintainable codebase following MVC + Service-Repository architecture.

7.2 Limitations

  • The system is limited to 101 food categories from the Food-101 dataset. Regional and traditional dishes may not be recognized.
  • Food images with multiple items on the same plate are not individually identified.
  • Portion size estimation is not implemented — nutrition data assumes a standard 100g serving.
  • The Hugging Face free API has rate limits (~30 requests/hour).

7.3 Future Scope

  • Multi-food Detection: Implement object detection (YOLO/Faster R-CNN) to identify multiple food items on a single plate.
  • Portion Estimation: Integrate depth sensing or reference object detection to estimate food quantity.
  • Expanded Dataset: Train the model on a larger dataset including Indian, Asian, and regional cuisines.
  • Mobile Application: Port the system to native Android/iOS using React Native or Flutter.
  • Barcode Scanning: Add barcode/QR code scanning for packaged food items.
  • Health Integration: Connect with wearable devices (Fitbit, Apple Watch) for holistic health tracking.
  • Doctor Dashboard: Enable healthcare professionals to monitor patient dietary patterns using the DOCTOR role.
  • Meal Planning: Generate complete daily/weekly meal plans based on nutritional goals and preferences.
